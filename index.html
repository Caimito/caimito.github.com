ATDD / BDD / TDD

Preventing defects from the start

Are you looking for a way to prevent defects? Then ATDD may be what you have been looking for. Stakeholders and your development team define together, in the form of executable specifications, what the software under development should do. Defects are prevented by continuos verification of production code against examples of correct behavior written in human language (English and others).

We put the focus on the expected behavior of the production code and we state that first before diving into the details. So that we get it right from the start.

By having tests in place for every piece of code we can trust our building blocks for the larger piece. Instead of believing that we may have done it right, we have proof at every turn of the way and can work with confidence. No sweaty hands before demos!

Activity-Centered Design

Solving the right problem

All too often simply the wrong software gets created. It may work but not as the users or stakeholders expected. Or it may solve the wrong problem altogether. Despite the vast improvements in the way how software teams create software since the Agile Manifesto was published in 2001 still many software projects fail in one way or another.

To many, the word design only describes how something looks but in fact it means how it works and looks. Well designed products are created based on a deep understanding of which activities users want to perform and are well engineered. This fitness makes these products superior to their competition and in some cases such products shake up an entire industry.

Activity-Centered Design helps to create smarter products that solve people's real problems by putting the focus on what people want to do through software instead of on how software is operated.

Learning Organization

Building on the agile mindset

Agile practices have helped software development teams to deliver higher quality code and better software solutions in numerous companies. However, the goal has never been to improve the efficiency of software production. Instead it is all about doing more effectively what people in organizations do. Regardless of whether that doing is making software or something else.

Learning organizations put an emphasis on inspect and adapt. They perform controlled experiments, measure thouroughly and then learn from the data for the next cycle. This has become known as the Lean Startup movement and increasingly is in use in medium and large enterprises.

The ability to learn quickly allows all kinds of organizations to adjust to fast paced change and meet the expectations of their customers - to be able to delight them.
